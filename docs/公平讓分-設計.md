# 公平讓分(進階開團功能)— 設計稿

> 狀態:**已實作並上線 2026-06-30**(預設值:寬限 N=4、門檻 X=2、取樣前 1/3、啟用最低 5 人;團主代排不受限)
> 目的:解決「手速快的人一直搶場地、打很多場,其他人偏少」的不公平。

## 1. 背景與問題
現況:到「排隊開放時間」後,臨打人可(a)直接點空的場上位置上場,或(b)排進候補(每場 4 格);一場結束後候補 **FIFO** 遞補。**誰手速快誰先佔**,跟打過幾場、休息多久無關 → 造成有人狂打、有人很少打。

90% 的團是「放任式自由上場」,團主不在乎也沒差。所以這是**進階、預設關、團主自由選擇**的功能,不影響多數團。

系統已記錄每位玩家 `games`(打過幾場)與 `total_minutes`,可直接用來算公平性。

## 2. 功能總覽(皆為開團層級、開團期間可即時調整、立即生效)

### ① 顯示場數(獨立開關)
- 團主決定要不要讓**臨打人前台**看到所有人的打球場數。
- 預設 **關**。
- **當「② 公平讓分」開啟時,本開關被無條件強制開啟**(公平模式必須透明,否則被擋的人不知道原因)。

### ② 公平讓分(獨立開關)
- 預設 **關**;團主**可隨時開/關**(例:打到最後 20 分鐘才開,讓收尾平均)。
- 可調參數(開團期間可即時改):
  - **寬限場數 N**(預設 `4`):每位玩家打滿 N 場**之前**完全自由(憑實力搶 OK),打滿才開始受限。
  - **門檻 X 場**(預設 `2`):定義「比平均多幾場」算超標。
- 固定參數(不開放團主調,降低心智負擔):
  - **平均取樣比例 = 前 1/3(最少 3 人)**。〔待確認〕

## 3. 精確規則(演算法)

**名詞**
- *在場有在輪的人(active)*:目前在場上 `playing` + 在候補 `queue` + **最近 20 分鐘**有上場過的玩家。排除早退/掛名單但走掉的人。
- *基準平均 avg*:把 active 的人依 `games` 由多到少排序,取**前 1/3(無條件進位,最少 3 人)**,算這群人的 `games` 平均。
- *門檻 threshold = avg + X*。

**封鎖判定(在「某人嘗試上場或排候補」的當下即時計算)**
```
若 ② 公平讓分 = 關        → 不限制(現行自由行為)
若 active 人數 < 5        → 不限制(人太少沒有公平問題,避免空轉)
否則,該玩家被擋,當且僅當:
    games(他) ≥ N            (已用完寬限)
  且 games(他) > avg + X       (嚴格大於;剛好等於不擋)
被擋 = 上場與排候補都不給,並回友善提示:
    「目前實施公平讓分,你已打 {games} 場(平均 {avg}),讓打較少的人先上,稍後再來」
```

**動態自我拉平(關鍵)**
- 每次有人嘗試加入都**用當下場數重算** avg / threshold,不是算一次定生死。
- 大家持續打 → avg 上升 → threshold 上升 → 被擋的人一旦不再 `> avg+X` 就**自動解鎖**。
- 範例(N=4, X=2):前 1/3 平均 10 → 門檻 12 → 你 13 被擋;之後平均升到 11 → 門檻 13 → 你 13 不再 >13 → 自動解鎖。

## 4. 邊界情況
- **人數太少(active < 5)**:不啟用限制(沒有公平問題,且避免場地空轉)。
- **前 1/3 不足 3 人**:取最少 3 人;若 active 本來就 < 3,等同上一條不限制。
- **剛好等於門檻**:不擋(用嚴格 `>`)。
- **寬限內(games < N)**:永遠不擋。
- **團主代排(現場排點板)**:團主手動排不受此限(團主有最終裁量權)。〔待確認:要不要也擋?預設不擋〕
- **關閉公平讓分**:立即恢復自由;「顯示場數」回到團主原本的設定值(若團主原本是關,就關回去)。

## 5. 後端設計
**Model(Session 新增欄位)**
- `ShowGames bool`(① 顯示場數開關)
- `FairPlay bool`(② 公平讓分開關)
- `FairGraceGames int`(N,預設 4)
- `FairThreshold int`(X,預設 2)

**API(團主,leader group,皆 `RequireSessionOwner`)**
- `PUT /sessions/:id/advanced` body `{ show_games, fair_play, fair_grace_games, fair_threshold }`(一次更新這組進階設定;開團期間可呼叫)
- 廣播 scope:`session`(只影響設定/顯示,不動 players/games 重抓)。但 `fair_play`/`show_games` 切換會影響前台呈現,前端收到後重抓 session 即可。

**計算位置**
- 在 `joinPlaying` / `joinQueue` 的 `bypassGate=false`(臨打人路徑)內,新增 `checkFairPlay(ctx, session, playerID)`。
- `checkFairPlay`:讀該場 players → 篩 active → 排序取前 1/3 → 算 avg → 套用第 3 節判定。
- 成本:每次 join 多一次「讀該場 players」(本來就常讀,可接受);只在 `fair_play=true` 時才算。
- 「最近 20 分鐘有上場過」需要 last-played 時戳:可用既有資料推估,或新增 `SessionPlayer.LastPlayedAt`(結束場地時更新)。〔實作時定〕

**回傳給前端的衍生資料**
- `SessionView` 增加 `fair_play`、`show_games`、(`fair_play` 時)`fair_avg`(即時平均)、`fair_threshold` 等,讓前端顯示。
- player 列表本來就有 `games`,前端依 `show_games` 決定顯不顯示。

## 6. 前端設計
**團主端(badminton-admin)**
- 開團進階設定區(設定 modal 內新增一張卡「⚙️ 進階:公平讓分」):
  - 兩個開關(顯示場數 / 公平讓分)+ N、X 數字調整。
  - **即時顯示目前算出來的平均**、門檻、目前被擋的人。
  - **白話規則說明**(摺疊):一段話講清楚「開啟後誰會被擋、為什麼、怎麼解除」。
  - 開公平讓分時,把「顯示場數」鎖成開、灰掉不可關,並註明原因。
- 開團建立表單**不放**這些(屬進行中微調),統一在管理頁設定處。

**臨打人端(badminton-booking)**
- `show_games` 開啟 → 場次頁顯示所有人目前場數(排行/標記)。
- 被擋時,上場/排候補按鈕給明確提示(見第 3 節文案)。
- 公平讓分開啟時,頁面顯示一個小標示「本場實施公平讓分」+ 點開看規則。

## 7. 預設值彙整
| 參數 | 預設 | 團主可調 |
|---|---|---|
| 顯示場數 | 關 | ✅(公平讓分開啟時強制開) |
| 公平讓分 | 關 | ✅(可隨時開關) |
| 寬限場數 N | 4 | ✅ |
| 門檻 X 場 | 2 | ✅ |
| 取樣比例 | 前 1/3(最少 3 人) | ❌ 固定〔待確認〕 |
| 啟用最低人數 | active ≥ 5 | ❌ 固定 |
